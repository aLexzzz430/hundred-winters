# 100 Winters design brief

## Visual thesis

A scientific field console carved into a living night map: charcoal, warm parchment, and one ember-orange signal turn a benchmark into a civilization under observation.

## Content plan

1. **Arena** — start inside the running protocol, with the world as the dominant visual and the selected agent's state beside it.
2. **Comparison** — make long-horizon divergence visible through one score field and a strict ranked ledger.
3. **Audit** — expose what the agent saw, what it chose, what the world changed, and what remained hidden.
4. **Protocol** — explain the causal world, fair shock schedule, scoring, Codex adapter, and reproduction commands.

## Interaction thesis

- A run advances season by season so the world visibly freezes, recovers, urbanizes, or degrades.
- Scrubbing the winter timeline morphs the same map and metrics instead of swapping disconnected cards.
- Switching agents moves one shared selection line across the ranked ledger and comparison curve, preserving spatial continuity.

## Product constraints

- The working benchmark is the first screen; there is no marketing hero in front of it.
- The browser demo is deterministic and keyless.
- Hidden judge state never enters the public replay payload.
- Motion must remain readable under `prefers-reduced-motion`.
- The full protocol must work on mobile, but the map remains the primary object on desktop.
